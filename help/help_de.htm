expandcontract_XH

Ein Plugin zum Expandieren und Kontrahieren für CMSimple_XH

Expandcontract_XH ermöglicht die Anzeige anderer CMSimple_XH-Seiten innerhalb einer Seite. Die Anzeige kann aufgeklappt (expand) und wieder geschlossen (contract) werden. Mehrere Aufrufe innerhalb einer Seite sind möglich. Mittels Parametern kann jeder Plugin-Aufruf abweichend von der Plugin-Konfiguration gestaltet werden.

Voraussetzungen

expandcontract_XH erfodert CMSimple_XH ≥ 1.6.3 und PHP ≥ 7.0

Installation

Die Installation erfolgt ebenso wie bei vielen anderen CMSimple_XH-Plugins.

  1. Zunächst sollten die Daten auf dem Server gesichert werden.
  2. Die heruntergeladene ZIP-Datei wird auf dem eigenen Computer entpackt.
  3. Das enthaltene Verzeichnis expandcontract/ wird jetzt komplett auf den Server in das plugins/-Verzeichnis von CMSimple_XH hochgeladen.
  4. Falls nötig, müssen Schreibrechte für die Unterverzeichnisse config/, css/ und languages/ und alle darin vorhandenen Dateien vergeben werden.

Einstellungen

Plugins → Expandcontract → Konfiguration

Hier können die Grundeinstellungen für das Plugin vorgenommen werden.

 

Use → Inline buttons Checkbox (default: aus)

Normalerweise (aus) werden die Links zum Aufklappen untereinander dargestellt und die aufklappenden Inhalte entfalten sich direkt unter dem jeweiligen Link.

Alternativ (ein) können die Links in einer Reihe nebeneinander als Buttons dargestellt werden. Die aufklappenden Inhalte entfalten sich dann unterhalb der Button-Reihe.

» Diese Einstellung kann beim Pluginaufruf mit dem Parameter show-inline verändert werden.

 

Use → Stylesheet Auswahlfeld (default: example1.css)

Das Plugin wird mit einigen Besipiel-CSS-Dateien für die Gestaltung der Links, der Buttons und der Expand-Bereiche ausgeliefert. Mit dem Auswahlfeld kann die gewünschte CSS-Datei gewählt werden. Bleibt das Auswahlfeld leer, dann werden nur grundlegende für die Funktionalität benötigte Styles aktiv – alle anderen Dinge (Links, Buttons usw.) werden ausschließlich über das CMS-Template beeinflusst.

Es ist auch möglich, eigene CSS-Dateien zu verwenden. Diese müssen lediglich im Plugin-CSS-Verzeichnis abgelegt werden, damit sie in der Auswahlliste erscheinen.

» Diese Einstellung gilt für alle auf der gesamten Website verwendeten Pluginaufrufe – sie kann nicht mittels Parameter verändert werden.

 

Expand-content → Padding Eingabefeld (default: leer)

Will man den expandierten Bereich z.B. mit einem Hintergrund oder einem Rahmen versehen, empfiehlt es sich, hier einen Randabstand (padding) einzutragen. Erlaubt sind alle validen CSS-Werte.

» Diese Einstellung kann beim Pluginaufruf mit dem Parameter content-padding verändert werden.

 

Expand-content → First open Checkbox (default: aus)

Beim Aufruf einer Seite kann der erste Aufklapp-Bereich automatisch geöffnet werden.

an = der erste vorgefundener Aufklapp-Bereich wird geöffnet

aus = alles ist geschlossen

» Diese Einstellung kann beim Pluginaufruf mit dem Parameter first-open verändert werden.

 

Expand-content → Auto close Checkbox (default: an)

Bestimmt, ob mehrere Aufklapp-Bereiche geöffnet bleiben können oder nicht.

an = wird ein Bereich geöffnet, werden alle anderen automatisch geschlossen

aus = es können beliebig viele Bereiche geöffnet werden

» Diese Einstellung kann beim Pluginaufruf mit dem Parameter auto-close verändert werden.

 

Expand-content → Show close button Checkbox (default: an)

Unterhalb des aufgeklappten Inhaltes kann ein Schließen-Button angezeigt werden.

an = der Schließen-Button wird angezeigt

aus = der Schließen-Button wird unterdrückt

» Diese Einstellung kann beim Pluginaufruf mit dem Parameter show-close verändert werden.

 

Expand-content → Max-height Eingabefeld (default: leer)

Die Höhe des aufklappenden Inhalts kann auf einen festen Wert begrenzt werden. Ist dieser Inhalt höher, dann wird der Inhalt scrollbar. Als Wert ist eine valide CSS-Höhenangabe erlaubt.

» Diese Einstellung kann beim Pluginaufruf mit dem Parameter max-height verändert werden.

Verwendung

Es gibt zwei Arten des Plugin-Aufrufes: ohne und mit Parametern.

 

Einfacher Plugin-Aufruf ohne Parameter:

{{{expand}}}

Das ist die einfachste Art des Pluginaufrufes. Dabei werden alle verborgenen Unterseiten der aufrufenden Seite angezeigt. Alle Einstellungen werden aus der Plugin-Konfiguration übernommen.

 

Plugin-Aufruf mit Parametern:

{{{expand 'Option1 = Wert(e)','Option2 = Wert(e)', … }}}

Bei einem Plugin-Aufruf mit Parametern, können die Standard-Einstellungen der Plugin-Konfiguration überschrieben werden. Die Parameter können in beliebiger Reihenfolge, getrennt durch Komma, aber immer in 'einfachen Anführungszeichen' übergeben werden. Alle Parameter sind optional – können also auch weggelassen werden.

Die einzelnen Parameter bestehen immer aus einem Bezeichner (Option) plus einem Gleichheitszeichen (=) plus einem Wert oder mehreren Werten. Bei mehreren Werten müssen diese ebenfalls durch Komma getrennt werden.

 

Die Verwendung der einzelnen Parameter wird ausführlich auf der Demo-Seite des Plugins erklärt. Dort werden auch die mitgelieferten CSS-Beispiele vorgestellt:
